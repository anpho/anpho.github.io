二、 拨号上网如何“突破”网速限制

首先需要明确一个核心概念：如果是运营商因为你触发 PCDN 策略而在局端（BRAS/SR 设备）对你的 PPPoE 会话进行了限速（例如上行被锁死在 5Mbps 或 10Mbps），这种限制在本地路由器端是绝对无法通过技术手段“突破”的。 限速发生在运营商机房，你的设备只能尝试“规避检测”以恢复正常带宽，或者优化物理网络。

1. 规避运营商 PCDN 检测（恢复被限制的上行带宽）

运营商检测 PCDN 主要依赖四个维度：上行流量饱和度、NAT 连接数、特定端口/SNI 特征、IPv6 滥用。你可以采取以下策略降低被标记的概率：

  • 物理隔离与 QoS 限速：在路由器上为跑 PCDN 的设备（或特定 IP/MAC）设置严格的 QoS。上行带宽绝对不能超过套餐标称上行的 50%（例如 30M 上行，限制在 15M 以内）。同时限制该设备的最大并发连接数（建议限制在 1000 以内）。
  • 关闭或严格限制 IPv6：大量 PCDN 调度利用 IPv6 绕过 NAT，导致连接数暴增，这是运营商目前查处的重灾区。建议在路由器中直接关闭 IPv6，或者在防火墙中严格限制 IPv6 的出站连接数。
  • 光猫改桥接，路由器拨号：光猫路由模式下的 NAT 类型和连接数管理较弱。将光猫改为桥接模式，使用性能较好的软路由或硬路由进行 PPPoE 拨号，利用路由器的 NAT 防火墙特性隐藏内网拓扑。
  • 重置会话（临时方案）：如果已经被临时限速，可以尝试在路由器中修改 WAN 口的 MAC 地址，然后重启光猫和路由器重新拨号。这有一定概率会更换 IP 或重置局端的限速策略，但治标不治本。

2. 突破物理/套餐带宽瓶颈（如果未被局端限速）

如果你是指宽带本身跑不满（例如 1000M 宽带只能跑到 300M），请排查以下拨号与硬件环节：

  • 光猫瓶颈：很多运营商下发的光猫 LAN 口只有百兆，或者光猫 CPU 性能不足无法处理千兆 NAT。必须改桥接，让路由器负责拨号和 NAT。
  • 多线多拨：如果当地运营商没有在局端做端口绑定或 MAC 地址绑定，可以使用支持多 WAN 的路由器进行“并发多拨”。通过负载均衡叠加带宽（注意：部分地区多拨会被系统自动限速或封号，需自行测试）。
  • 硬件链路排查：确保光猫到路由器、路由器到电脑的所有网线均为超五类（Cat5e）或六类（Cat6）以上；确保路由器的 WAN/LAN 口及电脑网卡支持千兆或 2.5G。
  • TCP 窗口与卸载：在软路由（如 OpenWrt/iKuai）中，开启硬件流量卸载（Flow Offloading），并适当调整 TCP 拥塞控制算法（如 BBR），可以提升高延迟下的吞吐量。

最后提示：目前三大运营商对家宽跑 PCDN 的打击力度空前，通常采用“第一次警告/限速 -> 第二次停机写承诺书 -> 第三次永久拆机并拉入黑名单”的策略。为了宽带的长期稳定使用，建议彻底清理内网中的 PCDN 设备。
